资讯 AI 分析 — 设计思路

一、模块定位

本模块是基金场景的延伸子模块，核心卖点是**"Doris 内置 AI Function 直接在数据库层完成文本智能分析"**。区别于传统"写入数据库 → 读出 → 调外部 AI → 再写回"的四步流程，Doris AI Function 在一条 SQL 内完成全链路：

```
原始资讯 → Stream Load 写入 Doris → SQL 调用 AI_SUMMARIZE/AI_SENTIMENT/AI_EXTRACT → 结果直接存储 → 可视化分析
```

定位不是资讯阅读器，而是**"AI Function 执行效果演示平台"**——让用户亲手点击三个 AI 函数按钮，直观看到 SQL 是什么、执行了什么、生成了什么结果，并对结果做多维分析对比。

二、Doris 技术亮点矩阵（核心卖点）

| Doris 特性 | 具体体现 | 展示方式 |
|---|---|---|
| AI_SUMMARIZE | 对金融资讯正文一键生成50字概括 | 按钮触发 + SQL 弹窗展示 + 结果卡片 |
| AI_SENTIMENT | 分析资讯情感（positive/negative/neutral/mixed） | 彩色情感标签 + 情感评分条 |
| AI_EXTRACT | 按5个标签提取结构化信息（JSON输出） | 标签云 + 结构化表格 + 频率统计 |
| DUPLICATE KEY 模型 | 每次 AI 处理写新行不更新 | QUALIFY 取最新行，多版本可追溯 |
| QUALIFY ROW_NUMBER() | 多版本数据去重取最新 | SQL 透明展示，性能说明 |
| 窗口函数聚合分析 | 情感分布、板块热力统计 | 实时图表，SQL 可视 |

三、数据表设计（1张核心表）

```sql
CREATE TABLE news_article (
    article_id      VARCHAR(10)   NOT NULL,   -- 文章ID（N001~N025）
    publish_ts      DATETIME      NOT NULL,   -- 发布时间（分钟级）

    -- 基础字段
    title           VARCHAR(300),             -- 标题
    content         TEXT,                     -- 正文（平均600字）
    source          VARCHAR(50),              -- 来源媒体
    sector_tag      VARCHAR(30),              -- 板块标签

    -- AI_SUMMARIZE 结果
    ai_summary      VARCHAR(500),             -- 生成摘要
    summarized      TINYINT       DEFAULT 0,  -- 是否已总结（0/1）

    -- AI_SENTIMENT 结果
    ai_sentiment    VARCHAR(20),              -- 情感类型
    sentiment_score INT,                     -- 情感评分(-100~100)
    sentiment_done  TINYINT       DEFAULT 0,  -- 是否已分析

    -- AI_EXTRACT 结果
    ai_extract      VARCHAR(2000),            -- JSON格式提取结果
    extracted       TINYINT       DEFAULT 0,  -- 是否已提取

    -- 元数据
    ai_method       VARCHAR(20)   DEFAULT 'PENDING'  -- DORIS_AI_FUNCTION / MOCK / PENDING
) Unique KEY(article_id, publish_ts)
DISTRIBUTED BY HASH(article_id) BUCKETS 4
PROPERTIES("replication_num"="1")
```

**为何用 DUPLICATE KEY 而非 UNIQUE KEY？**
每次调用 AI Function 后写入新行（带新的 publish_ts），旧行保留，形成处理历史。查询时始终使用 `QUALIFY ROW_NUMBER() OVER (PARTITION BY article_id ORDER BY publish_ts DESC) = 1` 取最新状态，同时可追溯历史版本。

五、三大 AI Function SQL 模板

**AI_SUMMARIZE — 文本高度概括**
```sql
-- Resource: qwen_llm（qwen-plus，QWEN provider，已在 Doris 中预配置）
UPDATE news_article
SET ai_summary = AI_SUMMARIZE('qwen_llm', content),
    summarized  = 1,
    ai_method   = 'DORIS_AI_FUNCTION'
WHERE summarized = 0
```

**AI_SENTIMENT — 情感倾向分析**
```sql
-- 返回值: positive | negative | neutral | mixed
UPDATE news_article
SET ai_sentiment  = AI_SENTIMENT('qwen_llm', content),
    sentiment_done = 1,
    ai_method      = 'DORIS_AI_FUNCTION'
WHERE sentiment_done = 0

-- 第二步：映射情感评分（-100~+100）
UPDATE news_article
SET sentiment_score = CASE ai_sentiment
    WHEN 'positive' THEN  70
    WHEN 'negative' THEN -70
    WHEN 'mixed'    THEN  15
    ELSE 0
END
WHERE sentiment_done = 1
```

**AI_EXTRACT — 结构化标签提取**
```sql
UPDATE news_article
SET ai_extract = AI_EXTRACT(
        'qwen_llm', content,
        ARRAY('事件类型','影响板块','关键政策或技术','核心公司','市场影响方向')
    ),
    extracted = 1,
    ai_method = 'DORIS_AI_FUNCTION'
WHERE extracted = 0
```
输出 JSON 示例：
```json
{
  "事件类型": "政策",
  "影响板块": ["半导体", "EDA"],
  "关键政策或技术": "EDA工具链28nm全流程验证",
  "核心公司": ["华大九天", "芯原股份"],
  "市场影响方向": "利好"
}
```

六、AI Function 调用方式

所有 AI 推理直接在 Doris SQL 层完成，Python 后端**不参与**任何 AI 调用，仅负责拼装 SQL 并通过 `execute_write()` 提交给 Doris：

```python
# Python 服务层伪代码（无任何 httpx / openai 调用）
rc = await execute_write("""
    UPDATE news_article
    SET ai_summary = AI_SUMMARIZE('qwen_llm', content),
        summarized  = 1,
        ai_method   = 'DORIS_AI_FUNCTION'
    WHERE summarized = 0
""")
```

Doris 在 FE/BE 内部通过 `qwen_llm` resource 访问 DashScope API，结果直接落盘到 `news_article` 表，`ai_method` 字段标记为 `DORIS_AI_FUNCTION`。

| Doris Resource | 参数 |
|---|---|
| Name | `qwen_llm` |
| provider_type | QWEN |
| model_name | qwen-plus |
| endpoint | https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions |

十、Doris 核心 SQL 示例

**取各板块最新情感分布：**
```sql
SELECT sector_tag,
       ai_sentiment,
       COUNT(*) AS cnt
FROM (
    SELECT article_id, sector_tag, ai_sentiment
    FROM news_article
    QUALIFY ROW_NUMBER() OVER (PARTITION BY article_id ORDER BY publish_ts DESC) = 1
    WHERE sentiment_done = 1
) t
GROUP BY sector_tag, ai_sentiment
ORDER BY sector_tag, cnt DESC
```

**情感评分分布直方图：**
```sql
SELECT FLOOR(sentiment_score / 10) * 10 AS score_bucket,
       COUNT(*) AS cnt
FROM (
    SELECT article_id, sentiment_score
    FROM news_article
    QUALIFY ROW_NUMBER() OVER (PARTITION BY article_id ORDER BY publish_ts DESC) = 1
    WHERE sentiment_done = 1
) t
GROUP BY score_bucket
ORDER BY score_bucket
```

**AI 处理进度统计（单次查询）：**
```sql
SELECT
    COUNT(DISTINCT article_id)          AS total,
    SUM(summarized)                     AS summarized,
    SUM(sentiment_done)                 AS sentiment_done,
    SUM(extracted)                      AS extracted
FROM (
    SELECT article_id, summarized, sentiment_done, extracted
    FROM news_article
    QUALIFY ROW_NUMBER() OVER (PARTITION BY article_id ORDER BY publish_ts DESC) = 1
) t
```

**全量 AI_EXTRACT（真实 Doris 生产写法）：**
```sql
UPDATE news_article

```

十一、核心亮点总结

1. **SQL-native AI** — AI 逻辑封装在 Doris SQL 内，无需外部 ETL，数据不出库
2. **三函数联动** — 摘要→情感→提取三步递进，结果在同一张表累积
3. **多版本追溯** — DUPLICATE KEY 保留处理历史，QUALIFY 取最新，不丢历史
4. **Demo 零依赖** — Python mock 策略确保无 API Key 时完整演示，切换真实 Key 后无缝升级
5. **结果可视化** — 不只是"调了 AI"，而是对 AI 输出做情感全景、标签云、板块热力多维分析，真正体现数据价值
