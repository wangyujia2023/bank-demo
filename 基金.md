基金数字沙盘 — 设计思路
一、模块定位
与智能制造一样，核心是**"动态演练 + Doris技术硬核展示"，不是静态报表。选取公募基金投研场景**，围绕"基金经理→持仓板块→实时估值→竞品对比"这条链路，把Doris最具说服力的特性全部压进来。

二、Doris 技术亮点矩阵（本模块核心卖点）
Doris特性	具体体现	展示方式
AGGREGATE MODEL + REPLACE_IF_NOT_NULL	IOPV高频刷新不写重复行	实时估值刷新，前端可见毫秒级更新
窗口函数 RANK() OVER PARTITION	同板块基金动态排名	竞品排名表，排名实时变动
CORR() 相关性聚合	多基金走势相关性矩阵	热力图，颜色驱动分仓建议
多表 JOIN 实时计算	持仓×行情→盘中估值	JOIN SQL透明展示
Stream Load 批量写入	模拟市场数据摄入	架构图动画+写入速率展示
物化视图	板块贡献预聚合	查询时间对比（有/无物化视图）

七、关键技术实现细节
AGGREGATE MODEL IOPV更新（无需DELETE+INSERT，直接流式更新）：

-- REPLACE_IF_NOT_NULL 保证高频更新只改实时字段，不影响其他聚合列
INSERT INTO fund_wide (fund_id, realtime_iopv, iopv_deviation, last_update_ts)
VALUES ('F001', 1.2345, 0.0234, NOW())
同板块排名（窗口函数）：

SELECT fund_id, fund_name, sharpe_ratio,
    RANK() OVER (PARTITION BY sector_tag ORDER BY sharpe_ratio DESC) AS sector_rank,
    RANK() OVER (PARTITION BY sector_tag ORDER BY max_drawdown ASC)  AS drawdown_rank
FROM fund_wide WHERE sector_tag = '半导体'
相关性矩阵（CORR聚合）：

SELECT a.fund_id, b.fund_id,
    ROUND(CORR(a.daily_return, b.daily_return), 3) AS corr
FROM fund_nav_history a JOIN fund_nav_history b ON a.trade_date = b.trade_date
WHERE a.fund_id IN (...) AND a.trade_date >= DATE_SUB(TODAY(), INTERVAL 90 DAY)
GROUP BY a.fund_id, b.fund_id
盘中IOPV实时估算（多表JOIN）：

SELECT p.fund_id,
    SUM(p.weight_pct/100 * q.current_price / q.prev_close) * f.nav_yesterday AS realtime_iopv
FROM fund_position p
JOIN stock_realtime q ON p.stock_code = q.stock_code
JOIN fund_wide f ON p.fund_id = f.fund_id
WHERE p.report_date = (SELECT MAX(report_date) FROM fund_position WHERE fund_id = p.fund_id)
GROUP BY p.fund_id, f.nav_yesterday
